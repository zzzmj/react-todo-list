## todo-list

这是一个简单的todo-list应用，之前写过几次，是为了打了一下基础知识，这次重写写这个应用主要是考虑设计思想，思路等

锻炼React思维方式

## 组件划分原则

1. 解耦：降低单一模块/组件的复杂度
2. 复用：保证组件一致性，提升开发效率
3. 组件颗粒度需要避免过大或过小
